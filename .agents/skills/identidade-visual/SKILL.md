# Skill de Identidade Visual

## Personalidade da interface

- A aplicacao deve parecer academica, clara e organizada.
- A prioridade e leitura rapida, acompanhamento de progresso e tranquilidade visual.
- Evite telas com excesso de elementos decorativos.

## Paleta

- Primaria: verde escuro `#205c49`.
- Secundaria: azul petroleo `#1f4e5f`.
- Destaque: amarelo suave `#f4c95d`.
- Fundo: cinza frio `#f5f7f8`.
- Texto principal: `#1f2933`.
- Texto secundario: `#52616b`.
- Erro: `#b42318`.
- Sucesso: `#16703c`.

## Tipografia

- Usar fontes de sistema para carregamento simples: `Inter`, `Segoe UI`, `Arial`, `sans-serif`.
- Titulos devem ser objetivos e menores em areas densas.
- Evitar texto em caixa alta fora de pequenos rotulos.

## Layout

- Usar largura maxima centralizada para paginas de conteudo.
- Preferir grids simples com boa area de toque.
- Em telas pequenas, formularios e acoes devem empilhar naturalmente.
- Cabecalho deve indicar sessao autenticada e oferecer navegacao essencial.

## Componentes

- Botoes primarios: fundo verde escuro e texto branco.
- Botoes secundarios: borda discreta e fundo branco.
- Badges de status devem ter cor sem depender apenas do texto.
- Formularios devem exibir erros proximo aos campos.
- Cartoes podem ser usados para entidades repetidas, com raio de ate 8px.

## UX

- Cada tela deve ter uma acao principal clara.
- Confirmar exclusoes destrutivas.
- Mensagens de sucesso devem ser visiveis apos criar, editar ou excluir.
- Listas vazias devem explicar o estado e oferecer acao de criacao.
